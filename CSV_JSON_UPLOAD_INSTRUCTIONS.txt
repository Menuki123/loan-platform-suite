CSV / JSON FILE UPLOAD - HOW TO RUN
===================================

WHAT IS NEW
-----------
1. Frontend now has an upload area for CSV or JSON files.
2. Agent reads the uploaded file and uses the file data during execution.
3. Evaluation result now shows:
   - uploaded file summary
   - API endpoints used by the agent
   - request dataset
   - response dataset
   - Show / Hide button
   - More button

BACKEND (Docker)
----------------
From the project root run:

  docker compose down
  docker compose up --build

Keep this terminal running.

FRONTEND (Local)
----------------
Open a second terminal:

  cd frontend
  npm install
  npm run dev

Then open:

  http://localhost:5173

LOGIN
-----
Username: admin
Password: password123

HOW TO TEST THE UPLOAD FEATURE
------------------------------
1. Login
2. Enter the Swagger URL and continue
   Example:
   http://localhost:3000/openapi.json

3. In the "Optional data upload" area, upload either:
   - a .csv file
   - a .json file

4. Type a prompt such as:
   Create a customer and apply for a loan

5. Click Send

6. The result card will now show:
   - Uploaded dataset used by the agent
   - API endpoints used by the agent
   - Route result
   - Reason
   - Request dataset
   - Response dataset

SUPPORTED FILE SHAPES
---------------------
CSV example:

customer_id,product_type,loan_amount,interest_rate,loan_term_months,loan_id,payment_amount,payment_method,reference_number
1,PRODUCT_A,5000,10,12,1,500,CARD,REF001

JSON example:

[
  {
    "customer_id": 1,
    "product_type": "PRODUCT_A",
    "loan_amount": 5000,
    "interest_rate": 10,
    "loan_term_months": 12,
    "loan_id": 1,
    "payment_amount": 500,
    "payment_method": "CARD",
    "reference_number": "REF001"
  }
]

IMPORTANT NOTES
---------------
1. The agent matches uploaded columns to endpoint payload fields.
2. If a field is missing in the uploaded file, the agent can still fall back to generated values.
3. This means the app now works with uploaded business data instead of only mock data.

IF THE FRONTEND DOES NOT LOAD
-----------------------------
Run inside frontend:

  rm -rf node_modules package-lock.json
  npm install
  npm run dev

If using Windows PowerShell:

  rmdir /s /q node_modules
  del package-lock.json
  npm install
  npm run dev
