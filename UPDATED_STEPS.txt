Updated flow included in this ZIP

What changed
1. Agent server now supports:
   - GET /qa/bootstrap
   - POST /qa/run with swaggerUrl or swaggerPath
2. Swagger loader now accepts:
   - remote Swagger/OpenAPI URLs
   - local relative file paths
3. Frontend now shows a guided wizard/chat flow:
   - Step 1: configuration card
   - Step 2: prompt input
   - Step 3: human-friendly results

How to run
1. Start the API server
   cd api-server
   npm install
   npm start

2. Start the agent server
   cd agent-server
   npm install
   npm start

3. Start the frontend
   cd frontend
   npm install
   npm run dev

Frontend default URL
- http://localhost:5173

Agent server default URL
- http://localhost:4000

API server default URL
- http://localhost:3000

How to use the new flow
1. Open the frontend
2. The page will automatically load the predefined config component from:
   GET http://localhost:4000/qa/bootstrap
3. Fill:
   - API Base URL
   - Swagger URL
   - Max Routes
4. Click Next
5. Enter the business/testing prompt
6. Click Send
7. Review the human-friendly execution summary

Example values
- API Base URL: http://localhost:3000
- Swagger URL: http://localhost:3000/openapi.yaml
  or a public URL such as your deployed swagger yaml/json endpoint
- Max Routes: 4

Important note
If the frontend build fails on your machine because of Vite/Rollup dependency issues, run:
   rm -rf node_modules package-lock.json
   npm install
or on Windows PowerShell:
   Remove-Item -Recurse -Force node_modules
   Remove-Item package-lock.json
   npm install
